# WhisperDesktop UI 改造计划（执行版）

## 目标
- 解决当前界面“工具感强、观感粗糙、层级不清”的问题。
- 保持核心工作流不变：选择输入 -> 选择模型/语言 -> 转写 -> 编辑 -> 导出。
- 降低用户理解成本，提升首次使用信任感。

## 视觉方向
- 风格：编辑工作台（paper + control board），避免“盗版工具感”的廉价深色主题。
- 色彩：暖底色 + 青绿色功能主色 + 橙色聚焦色（强调可操作状态）。
- 排版：标题采用衬线风格强调产品气质，正文用系统中文无衬线保证可读性。

## 信息架构重排
- 左侧控制区：输入源、模型与语言、动作区（下载模型/开始转写/状态）。
- 右侧结果区：统计信息（片段数/字符数/时长估算）+ 导出工具条 + 文本编辑区。
- 关键状态前置：将模型状态、转写进度、当前状态信息明确放在动作附近。

## 交互策略
- 拖拽区状态区分：默认/悬停/已有文件三态。
- 进度条支持确定/不确定两种动画状态。
- 导出行为不变，但视觉上靠近结果区，减少“跨区找按钮”。
- 文本噪声过滤默认启用，避免用户在预览区看到明显非口播标签。

## 转写文本质量优化
- 在后端分段结果进入前端前进行轻量清洗：
  - 过滤 `[music]`、`[applause]`、`（音乐）`、`【掌声】` 等常见非语音标签。
  - 保留正常语句，避免误伤真实内容。
- 该策略为默认启用，可后续扩展为开关选项。

## 验收标准
- 视觉：统一主题变量、层次明确、按钮状态清晰。
- 功能：转写、下载模型、导出 TXT/SRT 均与原逻辑一致。
- 兼容：桌面布局与窄屏（移动宽度）都可正常显示。
- 质量：Rust 单测通过、前端构建通过。
